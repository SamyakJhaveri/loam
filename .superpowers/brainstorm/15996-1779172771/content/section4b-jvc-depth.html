The Big Adaptation: Voice Architecture → Engineering AGENTS.md

JVC's three-file voice system (Constraint 05) maps elegantly to an engineering context:

JVC Voice Architecture (Content)
File 1: Voice & Tone (direction)
How you sound, teach, move through ideas. Patterns, not rules. Changes rarely.
File 2: Format Patterns (structure)
How each output type differs structurally. One paragraph per format. Grows slowly.
File 3: Constraints (never-do)
Hard boundaries. Cheapest in tokens, highest in impact. Changes often.
Loam v3.0 AGENTS.md (Engineering)
Section 1: Engineering Principles (direction)
How we approach problems: surgical changes, YAGNI, verify before claiming done, prefer existing patterns. ← karpathy-guidelines
Section 2: Output Patterns (structure)
Security practices, scalability patterns, model selection, context architecture, agent teams. ← dissolved skills
Section 3: Hard Constraints (never-do)
No giant CLAUDE.md, no skills without triggers, no building before using, no undocumented handoffs. ← JVC common mistakes

3 New JVC Integrations for v3.0

1. Session Update Discipline (from Constraint 04)
→ workflow.md §5 + /handoff

Every session ends with a 2-minute state update: what was done, what was decided, what's next. The /handoff skill enforces this structure. workflow.md §5 (Record) gets enhanced with the explicit three-bullet protocol.

2. Scaling Audit (from Constraint 07)
→ create-skill + layer-triage.md

Before creating a new skill, ask: Does this step require judgment? Would output be the same regardless of who does it? Is it a leverage point? The create-skill wizard gains a "scaling audit" pre-check that prevents over-automation. Layer-triage.md gains a "when to skill vs when to document" section.

3. Common Mistakes as Guardrails (from Example 04)
→ AGENTS.md §3 + known-issues.md

The 8 common mistakes become the Hard Constraints section of AGENTS.md. Engineering-adapted: no giant CLAUDE.md, always include Skip column, wire skills with triggers not availability, start minimal and grow from signals. These are persistent guardrails loaded every session.
