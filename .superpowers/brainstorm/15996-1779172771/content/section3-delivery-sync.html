Section 3: Delivery & Sync Workflow

How projects get bootstrapped and stay updated with Loam v3.0

Bootstrap Flow (New Project)

From zero to working project
$ uvx copier copy gh:samyakjhaveri/loam ./my-project

Copier asks 3 questions:
1. project_name: my-project
2. is_research: false
3. github_repo: owner/my-project (optional)

Copier renders seed/ → project root:
seed/.claude/ → .claude/
seed/CLAUDE.md.jinja → CLAUDE.md
seed/AGENTS.md.jinja → AGENTS.md
seed/README.md.jinja → README.md
seed/.gitignore.jinja → .gitignore
seed/.mcp.json.jinja → .mcp.json
seed/pyproject.toml.jinja → pyproject.toml

_tasks (post-copy):
If is_research → merge research/ overlay
Create seed dirs (scripts/, results/, config/...)
git init + initial commit
Optional: gh repo create

Update Flow (Pull Template Changes)

Downstream: template → project
$ cd my-project && uvx copier update

Copier does 3-way merge:
1. Compares old template (from .copier-answers.yml commit)
2. Against new template (latest tag)
3. Preserving your local changes

Result:
• New skills added automatically
• Updated rules merged (conflicts flagged)
• Your project-local skills untouched
• Your CLAUDE.md customizations preserved

Promote Flow (Push Project Innovation Back)

Upstream: project → template
$ /template-sync promote --layer generic .claude/skills/my-skill/

template-sync does:
1. Validates skill against agentskills.io schema
2. Scans for project-specific paths/secrets
3. Creates branch promote/my-skill in template repo
4. Copies skill to seed/.claude/skills/my-skill/
5. Commits + prints gh pr create command

⚠ Promote is opt-in, never automatic
⚠ Skill lands in template only after PR review + merge

What Changes from v2.0

v2.0 (Current)
  • 36 _exclude patterns in copier.yml
  • Every new file needs an exclusion rule
  • _research/ at root, merged by _tasks
  • Promote target unclear (seed/.claude/ vs .claude/)
  • Zero actual promotes in history
v3.0 (Proposed)
  • Zero exclusion patterns needed
  • New non-seed files are invisible by default
  • research/ at root, still merged by _tasks
  • Promote target clear: always seed/.claude/
  • Template-sync updated for seed/ path
