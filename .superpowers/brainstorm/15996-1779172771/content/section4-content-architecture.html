Section 4: Content Architecture

CLAUDE.md (L0) + AGENTS.md (persistent) + .claude/rules/ (L1/L2)

The Three-Layer Content Stack

Following JVC's ICM routing model, content lives in three layers with distinct roles and budgets.

CLAUDE.md — L0 (~800 tokens) — "Where am I?"
Always loaded. Map, not manual. Claude-specific features.

Contains:
  • What this project is (1-2 sentences)
  • Folder map (navigation table)
  • Reference doc index (.claude/rules/ pointers)
  • Workflow anchor (implement → review → validate → commit)
  • Claude-specific: permissions, hook behavior, session rules
Does NOT contain: instructions, patterns, checklists, architecture decisions
AGENTS.md — Persistent Context (~8KB) — "How do we work?"
Always loaded. Universal guidance. Works in Claude Code, Cursor, Copilot, Codex.

Sections (content from dissolved skills):
Coding Patterns
← karpathy-guidelines
Surgical changes, YAGNI, verify before claiming done, make assumptions explicit
Model Selection
← model-route
When to use Opus vs Sonnet, subagent allocation rules, cost awareness
Security Practices
← security skill
OWASP top 10 awareness, input validation, auth patterns, XSS/CSRF/SQLi prevention
Scalability Practices
← scalability skill
DB query patterns, caching, async processing, API design for scale
Context Architecture
← scaffold-context
When to add CONTEXT.md, the 6 required sections, the Skip column rule, sizing
Project Conventions
← new (project-specific)
Naming, file organization, commit style, PR conventions — each project fills this in
Why ~8KB? Vercel found this size range achieves near-perfect task success. Larger adds noise; smaller misses coverage.
.claude/rules/ — L1/L2 (~300 tok each) — "What do I do here?"
Loaded on demand. JVC routing rules. Path-scoped and task-scoped.

Always loaded (2):
  • workflow.md — 6-stage session workflow + anti-patterns
  • known-issues.md — active guardrails
On-demand JVC rules (5):
  • L0-budget.md — when editing CLAUDE.md
  • context-md-anatomy.md — when writing CONTEXT.md
  • stage-contract.md — when planning tasks
  • layer-triage.md — when scoping new tasks
  • validation-loop.md — when running /validate
Path-scoped (loaded by file type):
  • python.md — when editing .py files
  • frontend-design.md — when editing frontend files
  • architecture.md — when editing arch docs
  • tech-stack.md — when editing build/config files
