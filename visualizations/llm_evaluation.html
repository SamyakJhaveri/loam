Note on Speedup Ratio

The speedup ratio for bfs CUDA→OMP is ~0.001–0.002×, meaning OpenMP is ~500–1000× slower than CUDA for BFS. This is expected — BFS on GPU has massive parallelism advantages.

The speedup metric here measures translation correctness (does the LLM produce code that builds, runs, and passes verification?), not API performance. A ratio < 1.0 is a valid result: the LLM preserved correctness, but the target API is inherently slower for this workload on this hardware.

Timing method note: Claude bfs used cpu_time (via GNU time, user+sys=0.83s); Azure bfs used wall_time (0.482s). The baseline is always wall_time from the spec, creating a slight apples-to-oranges comparison. Re-running all baselines with CPU timing is a future task.
