Known Pipeline Issues

backup_files() / restore_files() crash corruption
If the harness crashes between backup_files() and restore_files(), reference source files in the original source directory can be left in a corrupted (augmented) state. The sibling temp-directory approach in augment_verify.py avoids this, but the eval pipeline's llm_evaluate.py still uses backup/restore.
Multi-file extraction: LLM cannot produce subdir-pathed files (M11)
When a benchmark has source files in subdirectories, the LLM's code-fence output is extracted as flat files. There is no mechanism for the LLM to specify subdirectory paths in its response. This affects multi-file benchmarks like backprop. Tracked as M11 concern.
gaussian-omp spec broken
The rodinia-gaussian-omp spec is currently non-functional and should be excluded from batch evaluation runs.
