Fix V: _fatal_diagnostics threshold
Changed from severity >= Fatal (4) to severity >= Error (3). This catches real errors that were previously missed, preventing broken transforms from passing validation.
Fix A: Overlap filter
Added _edits_overlap() check between selected candidates + merged re-validation. Prevents nested subscripts (e.g., J[iS[i]*cols+j]) from producing corrupted output.

Applied Bug Fixes

Fix V — _fatal_diagnostics threshold
Before: diag.severity >= ci.Diagnostic.Fatal (value 4)
After: diag.severity >= ci.Diagnostic.Error (value 3)
Effect: Catches compile errors that were previously invisible to validation
Fix A — Overlapping nested subscripts
Problem: J[iS[i]*cols+j] — both inner and outer selected, edits corrupt bytes
Fix: _edits_overlap() filter + merged RewriteCandidate re-validation
Fix B — Struct member precedence
Problem: arr[i].member → *((arr)+(i)).member — . binds tighter
Fix: Detect next char is ., emit (*((...))) with extra parens
Fix C — Assignment in condition
Problem: fp = fopen(...) == 0 → swap produces non-lvalue
Fix: _contains_assignment() check skips comparisons with assignments
Bug D — TypedefExpansion struct pointers (UNFIXED)
TypedefExpansion replaces Point* with the underlying struct type but doesn't handle the pointer declarator correctly. Breaks streamcluster-{cuda,omp} at L2+.
