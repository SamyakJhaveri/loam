The 5 Transforms — Status

#TransformWhat It DoesStatusNotes
1 ArithmeticTransform x += 1 ↔ x = x + 1 ✓ Safe Simple operator expansion; no edge cases observed
2 SwapCondition a < b → b > a ✓ Safe Bug C fixed (2026-03-11): skip comparisons where operand contains assignment
3 PointerArith→Array *(arr+i) ↔ arr[i] ✓ Mostly safe Bug A (overlap) + Bug B (member access precedence) fixed. Bug F/G remain at high edit-site counts
4 TypedefExpansion Expands typedef aliases to underlying types ⚠ Bug D Expands struct/pointer typedefs to non-compilable types. Fix: filter in _find_candidates(). ~1 hr
5 ChangeNames Renames local variables ⚠ Bug E (L3/L4) Renames struct field references by token scan — doesn't distinguish var.field. Fix: skip tokens after ./->. ~2 hr

Already Fixed (2026-03-11)

BugDescriptionFix Applied
Bug A ✓ PointerArith: overlapping nested subscripts corrupt byte offsets Added overlap filter + re-validation of merged candidate in AstTransform.apply()
Bug B ✓ PointerArith: arr[i].member → wrong pointer precedence Emit (*((arr)+(i))) when next char is .
