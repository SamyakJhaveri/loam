# External Integrations

**Analysis Date:** 2026-04-03

## LLM APIs

All LLM API calls are routed through `scripts/evaluation/llm_evaluate.py` via the
`call_llm()` function (line 730). Provider selection is based on model ID prefix.

**Anthropic (Claude):**
- Models: `claude-sonnet-4-20250514`, `claude-sonnet-4-6-20260218`, `claude-opus-4-6-20260205`, `claude-haiku-4-5-20251001`
- SDK: `anthropic` Python package (v0.85.0)
- Auth: `ANTHROPIC_API_KEY` environment variable
- Endpoint: Default Anthropic API endpoint (SDK handles URL)
- Config: `max_tokens=32768`, `temperature=0.0`, no `thinking` parameter (extended thinking disabled for parity)
- Notes: Primary eval model is `claude-sonnet-4-20250514`

**OpenAI:**
- Models: `gpt-4o`, `gpt-4.1-2025-04-14`, `o3-2025-04-16`, `o4-mini-2025-04-16`
- SDK: `openai` Python package (v2.28.0)
- Auth: `OPENAI_API_KEY` environment variable
- Config: `max_completion_tokens=32768`; reasoning models (`o1-*`, `o3-*`, `o4-*`) do not receive `temperature`

**Azure OpenAI:**
- Model: `azure-gpt-4.1` (deployment name: `gpt-4.1`, stripped from prefix)
- SDK: `openai.AzureOpenAI` client
- Auth: `AZURE_OPENAI_API_KEY` + `AZURE_OPENAI_ENDPOINT` environment variables
- API version: `2025-01-01-preview`
- Notes: "azure-" prefix is stripped to get deployment name. Endpoint URL parsed to strip path/query.
- PBS job: `AZURE_OPENAI_ENDPOINT="https://galor-m8yvytc2-swedencentral.cognitiveservices.azure.com/"`

**Groq (OpenAI-compatible):**
- Model: `groq-llama-3.3-70b-versatile` (API model: `llama-3.3-70b-versatile`)
- SDK: `openai.OpenAI` with `base_url="https://api.groq.com/openai/v1"`
- Auth: `GROQ_API_KEY` environment variable
- Config: `max_tokens=32768`
- Limitation: 16384 completion token cap; large multi-file kernels (myocyte, heartwall) hit cap

**Google Gemini (OpenAI-compatible):**
- Models: `gemini-2.5-flash-lite`, `gemini-2.5-flash`
- SDK: `openai.OpenAI` with `base_url="https://generativelanguage.googleapis.com/v1beta/openai/"`
- Auth: `GEMINI_API_KEY` or `GOOGLE_API_KEY` environment variable
- Config: `max_tokens=65536`, `reasoning_effort="none"` (explicitly disables thinking)
- Notes: Flash Lite has thinking off by default; `reasoning_effort="none"` is belt-and-suspenders

**Together AI (OpenAI-compatible):**
- Model: `together-qwen-3.5-397b-a17b` (API model: `Qwen/Qwen3.5-397B-A17B`)
- SDK: `openai.OpenAI` with `base_url="https://api.together.xyz/v1"`
- Auth: `TOGETHER_API_KEY` environment variable
- Config: `max_tokens=81920`, `extra_body={"chat_template_kwargs": {"enable_thinking": False}}`
- Notes: Thinking disabled via Jinja2 template kwargs passthrough; Qwen 3.5 has thinking ON by default

## Data Storage

**Databases:**
- None. No SQL, NoSQL, or embedded database used.

**Persistent stores:**
- `manifest.jsonl` — Append-only JSONL registry of all kernel specs. Never modify existing entries.
- `results/evaluation/{model}/*.json` — Immutable per-task evaluation result JSONs (one file per translation task)
- `results/evaluation/eval_summary.json` — Aggregated results; regenerated by `scripts/evaluation/analyze_eval.py`
- `results/augmentation/*.json` — Augmentation pipeline result records
- `results/analysis/sloc_analysis.json` — SLoC characterization of benchmark corpus

**File Formats:**
- Spec files: `specs/{suite}-{kernel}-{api}.json` — JSON conforming to `schema/spec_schema.json` (spec_version 1.0.0)
- Manifest: `manifest.jsonl` — JSONL, one entry per spec, conforming to `schema/manifest_schema.json`
- Result JSONs: Flat JSON with fields: `overall_status`, `source_spec`, `target_spec`, `model`, `prompt_tokens`, `completion_tokens`, `attempts[]`, `speedup_ratio`, `timing_method`, `translation_type`, `run_args_mode`, `verification_mode`
- Dashboard data: `visualizations/eval_results_data.js` — JavaScript blob loaded by browser dashboard

**File Storage:**
- Local filesystem only. All benchmark sources, specs, and results are on disk.
- Benchmark source repos as git submodule: `rodinia/rodinia-src/` (commit `9c10d3ea`)
- HeCBench: `HeCBench-master/` — gitignored, not cloned locally on development machine

## Authentication & Identity

**Auth Provider:**
- No application-level auth. This is a research tool run locally and on HPC clusters.
- API keys for LLM providers are passed as environment variables (never committed to repo).

**Required environment variables:**
- `ANTHROPIC_API_KEY` — Claude models
- `OPENAI_API_KEY` — OpenAI models (gpt-4o, etc.)
- `AZURE_OPENAI_API_KEY` — Azure OpenAI
- `AZURE_OPENAI_ENDPOINT` — Azure OpenAI endpoint URL
- `GROQ_API_KEY` — Groq / Llama models
- `GEMINI_API_KEY` or `GOOGLE_API_KEY` — Google Gemini models
- `TOGETHER_API_KEY` — Together AI / Qwen models

## Monitoring & Observability

**Error Tracking:**
- No external error tracking service.
- Build errors captured in `build_error_snippet` field of result JSONs (head+tail of stderr, max 1500 chars)
- Run errors captured in `run_stderr_snippet` and `run_stdout_snippet` fields

**Logs:**
- Python `logging` module throughout harness and eval pipeline; `-v` flag enables verbose output to stderr
- Batch scripts write per-kernel logs to `results/phase*/` and `results/evaluation/` directories
- Long-running evals use tmux for session persistence; log files named with timestamps

**Metrics captured per evaluation task:**
- `prompt_tokens`, `completion_tokens` — LLM token usage
- `duration_seconds` — LLM call wall time
- `finish_reason` — `"stop"` (complete) or `"length"` (truncated)
- `speedup_ratio`, `timing_method` — Performance metrics (wall-clock only; see CONCERNS)

## CI/CD & Deployment

**Hosting:**
- Local Linux machine (NVIDIA RTX 4070, AMD Ryzen 9 7900X)
- ALCF Polaris HPC cluster (for production eval campaigns via PBS job `run_eval_campaign.pbs`)

**CI Pipeline:**
- No external CI service (no GitHub Actions, no Jenkins).
- Pre-commit hook: enforces `.validation_passed` sentinel before `git commit`
- PostToolUse hook: `ruff check` on all Python files edited

**Docker:**
- `Dockerfile` provides CPU-only validation image for schema checks and unit tests
- Build: `docker build -t parbench .`
- Does NOT include CUDA/GPU support — only for validation, not eval runs

## External Tools Invoked at Runtime

**Build system — invoked by `harness/builder.py` via `subprocess`:**
- `make` — primary build tool for all benchmark suites
- `nvcc` — CUDA compiler at `/opt/nvidia/hpc_sdk/Linux_x86_64/24.3/cuda/bin/nvcc`
- `g++` / `gcc` — OpenMP and OpenCL builds
- `nvc` / `nvc++` — OpenMP target offload builds (NVIDIA HPC SDK 24.3)

**Validation — invoked by `scripts/validate_schema.py`:**
- `jsonschema` Python library for spec/manifest schema validation

**Augmentation — invoked by `c_augmentation/`:**
- `libclang` Python bindings — AST parsing for C/C++/CUDA/OpenCL transforms
