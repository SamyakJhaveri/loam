# External Integrations

**Analysis Date:** 2026-04-09

## APIs & External Services

**LLM Providers (for code translation evaluation):**

All LLM integrations are in `scripts/evaluation/llm_evaluate.py`, dispatched by model name prefix in `_call_llm()`.

| Provider | Model Prefix | SDK | Auth Env Var | Base URL |
|----------|-------------|-----|-------------|----------|
| Anthropic | `claude-*` | `anthropic` (native) | `ANTHROPIC_API_KEY` | Default (api.anthropic.com) |
| OpenAI | `gpt-*`, `o1-*`, `o3-*`, `o4-*` | `openai` (native) | `OPENAI_API_KEY` | Default (api.openai.com) |
| Azure OpenAI | `azure-*` | `openai.AzureOpenAI` | `AZURE_OPENAI_API_KEY` + `AZURE_OPENAI_ENDPOINT` | Per-deployment endpoint |
| Groq | `groq-*` | `openai` (compatible) | `GROQ_API_KEY` | `https://api.groq.com/openai/v1` |
| Google AI | `gemini-*` | `openai` (compatible) | `GEMINI_API_KEY` or `GOOGLE_API_KEY` | `https://generativelanguage.googleapis.com/v1beta/openai/` |
| Together AI | `together-*` | `openai` (compatible) | `TOGETHER_API_KEY` | `https://api.together.xyz/v1` |

**Registered Models (in `MODEL_REGISTRY` dict):**
- `claude-sonnet-4-20250514`, `claude-sonnet-4-6-20260218`, `claude-opus-4-6-20260205`, `claude-haiku-4-5-20251001`
- `gpt-4o`, `gpt-4.1-2025-04-14`, `o3-2025-04-16`, `o4-mini-2025-04-16`
- `azure-gpt-4.1`
- `groq-llama-3.3-70b-versatile`
- `gemini-2.5-flash-lite`, `gemini-2.5-flash`
- `together-qwen-3.5-397b-a17b` (API model: `Qwen/Qwen3.5-397B-A17B`)

**Key integration patterns:**
- All non-Anthropic providers use the OpenAI SDK with custom `base_url`
- Max completion tokens: 32768 (most), 65536 (Gemini), 81920 (Together/Qwen)
- Temperature: configurable, default varies by use case
- Thinking/reasoning explicitly disabled for all models (fair comparison)
- Together AI uses `extra_body.chat_template_kwargs.enable_thinking=False` for Qwen
- Gemini uses `reasoning_effort="none"`

## CI/CD & Deployment

**GitHub Actions:**
- Single workflow: `.github/workflows/deploy-pages.yml`
- Purpose: Deploy `visualizations/` directory to GitHub Pages
- Trigger: Push to `main` touching `visualizations/**`, or manual `workflow_dispatch`
- Runner: `ubuntu-latest`
- Steps:
  1. Checkout code
  2. Install `staticrypt` via npm
  3. Encrypt all HTML files with `PAGES_PASSWORD` secret (AES-256)
  4. Upload `visualizations/` as Pages artifact
  5. Deploy to GitHub Pages

**No CI for:**
- Python tests (run manually: `pytest`)
- Schema validation (run manually: `python3 scripts/validate_schema.py --all`)
- LLM evaluations (run manually on GPU machine via tmux)
- Paper compilation (run manually via `make` in `docs/paper/latex/`)

**Hosting:**
- GitHub Pages: `https://samyakjhaveri.github.io/parbench_sam/`
  - Static HTML dashboard (7 HTML files + 2 JS data files + CSS)
  - Password-protected via staticrypt
  - Entry point: `visualizations/index.html` redirects to `overview.html`

**Local Execution:**
- All LLM evaluations run on the Linux GPU machine (not cloud-hosted)
- Long-running eval batches managed via tmux sessions
