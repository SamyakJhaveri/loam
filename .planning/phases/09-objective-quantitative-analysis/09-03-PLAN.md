Implement the `run_validation()` function in `scripts/analysis/quantitative_findings.py` (per D-37 through D-41). This function is called when `--validate` is passed. It takes the computed output dict and project_root, performs 4 categories of checks, prints results to stdout, and writes to `results/analysis/quantitative_findings_validation.json`.

**Validation structure:**
```python
def run_validation(output: dict, project_root: Path, verbose: bool) -> dict:
    results = {
        "timestamp": datetime.now(timezone.utc).isoformat(),
        "spot_checks": [],      # Independent verification of key numbers
        "cross_checks": [],     # Comparison with existing analysis files
        "consistency_checks": [], # Internal consistency of the output
        "paper_claims_audit": [], # Pre-audit of paper.tex claims
        "summary": {"total": 0, "passed": 0, "failed": 0, "warnings": 0}
    }
    # ... run all checks ...
    return results
```

**Category 1: Spot-checks** (per D-37 -- 10+ checks using alternative code paths):

Each spot-check independently counts raw files to verify a computed value:

1. **Total file count**: `len(list(results_dir.glob("*.json")))` minus batch/summary files == `output["metadata"]["file_counts"]["total_on_disk"]`
2. **Campaign 1 count**: Count files where `json.load(f).get("temperature", 0) == 0.0` after KNOWN_FAIL exclusion == `output["metadata"]["file_counts"]["campaign_1_valid"]` (700)
3. **Campaign 2 count**: Count files where temperature > 0 after KNOWN_FAIL exclusion == `output["metadata"]["file_counts"]["campaign_2_valid"]` (420)
4. **KNOWN_FAIL exclusion count**: Count files where source_spec or target_spec is in EXCLUDED_SPECS == `output["metadata"]["file_counts"]["excluded_known_fail"]` (128)
5. **Rodinia C1 pass count**: For Rodinia L0 C1 files, count those with `overall_status == "PASS"` == aggregate_pass_rates.per_suite.rodinia.value * rodinia.n (approximately, within rounding)
6. **Overall C1 pass rate**: PASS count / total C1 valid == aggregate_pass_rates.overall.value (within 0.001)
7. **Direction count check**: Number of distinct directions in C1 L0 data == len(standard) + len(case_study) directions
8. **Per-suite file count sum**: Sum of per-suite C1 counts == total C1 valid (700)
9. **Augmentation level distribution**: Count of L0 + L1 + L2 + L3 + L4 files in C1 == total C1 valid (700)
10. **Campaign 2 seed count**: For each task in C2, verify exactly 3 seeds (group by source_spec+target_spec, check len==3)
11. **BUILD_FAIL count**: Count C1 records with overall_status=="BUILD_FAIL" == failure_taxonomy.aggregate.BUILD_FAIL.count

For each spot-check, record: `{"check": "name", "expected": X, "actual": Y, "status": "PASS"/"FAIL", "tolerance": 0.001}`.

**NOTE:** Spot-checks use a DIFFERENT code path than the main computation. Don't call the existing functions -- independently load and count files using simple loops. This catches bugs in the main computation logic.

**Category 2: Cross-checks** (per D-38 -- comparison with existing analysis files):

Compare computed values against existing analysis JSONs. Warn on discrepancies > 0.1%. NEVER fail on discrepancies -- warn only (per D-38).

1. **vs paper_data.json**: Compare primary_campaign.overall pass rate, per-direction rates
2. **vs paper_data_rodinia.json**: Compare Rodinia-only aggregate pass rate
3. **vs statistical_analysis.json**: Compare Wilson CIs, Cochran-Armitage z/p values
4. **vs error_taxonomy.json**: Compare failure type counts
5. **vs selfrepair_analysis.json**: Compare overall repair rate
6. **vs token_analysis.json**: Compare total tokens, total cost

For each cross-check: `{"check": "name", "our_value": X, "their_value": Y, "discrepancy_pct": Z, "status": "MATCH"/"WARNING"/"DIFFERENT_SCOPE", "note": "..."}`.

**NOTE:** Many discrepancies are expected because of different KNOWN_FAIL exclusion sets (6 vs 8 specs) and different scope (Rodinia-only vs all-suite). Use "DIFFERENT_SCOPE" status for these. Only flag "WARNING" for same-scope comparisons with >0.1% discrepancy.

**Category 3: Internal consistency checks** (per D-39):

1. For every wilson_ci result: assert ci_lower <= value <= ci_upper
2. Per-suite counts sum to aggregate total: sum(per_suite.*.n) == overall.n
3. No NaN or null in required numeric fields (recursive check on output dict)
4. Augmentation level counts: L0 + L1 + L2 + L3 + L4 file counts match C1 total
5. pass@k: pass_at_1.value >= pass_at_3.value (any passing >= all passing)
6. Token cost: total_cost == input_cost + output_cost (within rounding)
7. Failure taxonomy: sum of failure type counts + PASS count == total C1 records

For each: `{"check": "name", "status": "PASS"/"FAIL", "detail": "..."}`.

**Category 4: Paper claims pre-audit** (per D-40):

Parse `docs/paper/latex/paper.tex` for each claim in `output["paper_claims"]`:
- For each claim, search paper.tex for the expected value pattern (e.g., "36.2" for a 36.2% claim)
- Report whether the claim's current_value matches what's found in paper.tex
- Status: "MATCH" (paper has the right value), "STALE" (paper has a different value), "NOT_FOUND" (value not found in paper)

For each: `{"claim_id": "...", "paper_value": "...", "findings_value": "...", "status": "MATCH"/"STALE"/"NOT_FOUND", "paper_location": "..."}`.
