Write a concise subsection (approximately 15-25 lines of LaTeX):

```latex
\subsection{Prompt Anonymization}
\label{sec:anonymization}

To minimize training-data leakage through prompt content, ParBench applies six anonymization measures to every translation prompt before submission to the LLM:

\begin{enumerate}
  \item \textbf{Kernel identity removal.} The kernel name and description are omitted entirely from the prompt; the system message provides only the translation task specification without identifying the benchmark kernel.
  \item \textbf{Comment stripping.} All source code comments are removed via an AST-aware stripping pass, eliminating author attributions, algorithm descriptions, and benchmark-specific annotations that could trigger memorized associations.
  \item \textbf{Filename genericization.} Source filenames are replaced with generic identifiers (``Source File 1'', ``Source File 2'') to prevent the model from inferring the kernel from file naming conventions.
  \item \textbf{Target filename anonymization.} Target output filenames use a generic scheme (\texttt{translated\_0.ext}, \texttt{translated\_1.ext}) rather than kernel-specific names.
  \item \textbf{Build command anonymization.} Kernel names in build commands (e.g., make targets) are replaced with generic placeholders.
  \item \textbf{Support file genericization.} Header and auxiliary files are renamed to generic identifiers (``Header File 1'', ``Code File 1'') in the prompt.
\end{enumerate}

These measures complement the AST-driven augmentation engine (Section~\ref{sec:augmentation-robustness}): augmentation tests whether surface-level code changes affect translation quality, while anonymization prevents the model from identifying the benchmark kernel and recalling memorized translations.
```
