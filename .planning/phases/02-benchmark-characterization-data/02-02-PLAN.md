    Create `scripts/analysis/test_benchmark_characterization.py` with pytest tests organized in sections.

    **Test architecture:** Tests derive expected values INDEPENDENTLY from raw data sources (manifest.jsonl, spec files, source dirs), then compare against the script's output. This ensures the script is validated against ground truth, not circular self-validation.

    **Header and imports:**
    ```python
    """Tests for benchmark_characterization.py -- validates output against independently derived ground truth.

    Tests verify each characterization metric by computing expected values directly
    from raw data sources (manifest.jsonl, spec JSONs, source directories) and
    comparing against the script's output. This catches bugs via independent computation.
    """
    import json
    import subprocess
    import sys
    from pathlib import Path
    import pytest

    PROJECT_ROOT = Path(__file__).resolve().parent.parent.parent
    SCRIPT_PATH = PROJECT_ROOT / "scripts" / "analysis" / "benchmark_characterization.py"
    OUTPUT_JSON = PROJECT_ROOT / "results" / "analysis" / "benchmark_characterization.json"
    MANIFEST_PATH = PROJECT_ROOT / "manifest.jsonl"
    SPECS_DIR = PROJECT_ROOT / "specs"
    ```

    **Fixture:**
    ```python
    @pytest.fixture(scope="module")
    def char_data() -> dict:
        """Load benchmark_characterization.json. Skip if not yet generated."""
        if not OUTPUT_JSON.exists():
            pytest.skip("benchmark_characterization.json not yet generated -- run benchmark_characterization.py first")
        return json.loads(OUTPUT_JSON.read_text())
    ```

    **Test sections (one per CHAR requirement):**

    **Section 1: End-to-end script execution**
    - `test_script_runs_successfully()`: Run script via `subprocess.run([sys.executable, str(SCRIPT_PATH), "--project-root", str(PROJECT_ROOT)])`, assert returncode == 0, assert OUTPUT_JSON.exists(), assert output is valid JSON.

    **Section 2: CHAR-01 SLoC tests**
    - `test_sloc_covers_all_35_kernels(char_data)`: Assert `len(char_data["sloc"]["per_kernel"]) == 35`
    - `test_sloc_has_required_fields(char_data)`: For each kernel in sloc.per_kernel, assert keys include: `kernel`, `suite`, `physical_sloc`, `num_source_files`, `num_target_files`. Assert `physical_sloc > 0` for each.
    - `test_sloc_summary_statistics(char_data)`: Assert summary has `min_sloc`, `max_sloc`, `mean_sloc`, `median_sloc`. Assert `min_sloc > 0` and `max_sloc > min_sloc`.
    - `test_sloc_omp_ratio_present(char_data)`: For kernels with OMP specs, assert `omp_cuda_ratio` is a positive float. (D-04 says compute ratio.)
    - `test_sloc_kernel_names_match_corpus(char_data)`: Import CORPUS_KERNELS from sloc_analysis, verify set of kernel names in output matches set of kernel names in CORPUS_KERNELS.

    **Section 3: CHAR-02 category tests**
    - `test_categories_has_12_entries(char_data)`: Assert `len(char_data["categories"]) == 12`
    - `test_category_names_correct(char_data)`: Assert set of keys == `{"crypto", "financial", "graph", "image", "linear_algebra", "ml", "molecular_dynamics", "other", "physics", "reduction", "sort", "stencil"}`
    - `test_categories_have_suite_annotations(char_data)`: For each category, assert `"suites"` key exists and is a dict (per D-05). Assert `"kernel_count"` exists and is > 0.
    - `test_category_counts_match_manifest(char_data)`: Independently load manifest.jsonl, filter phantoms, count distinct kernels per category, compare against char_data["categories"]. Counts must match exactly.

    **Section 4: CHAR-03 API coverage tests**
    - `test_api_coverage_has_5_suites(char_data)`: Assert `len(char_data["api_coverage"]["suites"]) == 5`
    - `test_api_coverage_has_4_apis(char_data)`: For each suite, assert keys include `cuda`, `omp`, `opencl`, `omp_target` (per D-06).
    - `test_api_coverage_matches_specs(char_data)`: Independently count distinct kernel_names per (suite, api) from manifest.jsonl (filtered), compare against cross-tab values. Must match exactly.
    - `test_api_coverage_totals_consistent(char_data)`: Sum each column across suites, compare against totals dict.

    **Section 5: CHAR-04 multi-file tests**
    - `test_multi_file_covers_all_kernels(char_data)`: Assert `len(char_data["multi_file"]["by_kernel"]) == 35`
    - `test_multi_file_has_both_counts(char_data)`: For each kernel, assert both `prompt_payload_count` and `translation_targets_count` keys exist (per D-02).
    - `test_multi_file_classification_uses_targets(char_data)`: Verify `is_multi_file` == (`translation_targets_count > 1`) for each kernel. This enforces D-02.
    - `test_multi_file_aggregate_sums(char_data)`: Assert `aggregate.single_file_count + aggregate.multi_file_count == 35`.
    - `test_multi_file_matches_specs()`: For 3 specific kernels (bfs=3 targets, nn=1 target, xsbench=2 targets), independently load spec and verify counts match.

    **Section 6: CHAR-05 language features tests**
    - `test_language_features_has_entries(char_data)`: Assert `"per_kernel"` key in language_features, assert len > 0 (at least some kernels have features).
    - `test_language_feature_tiers_valid(char_data)`: For each kernel with features, assert `tier` is one of: `"cuda_basic"`, `"cuda_library"`, `"cuda_9plus"`, `"omp_basic"`, `"omp_4.5"`, `"omp_target"`, `"opencl_1x"`, `"opencl_2x"`, `"undetected"`.
    - `test_bfs_has_cuda_basic_features(char_data)`: BFS CUDA source uses `__global__` — assert tier is at least "cuda_basic".
    - `test_language_features_skip_missing_dirs(char_data)`: Kernels with missing source dirs should be noted but not crash the script.

    **Section 7: CHAR-06 language standards tests**
    - `test_language_standards_distribution_sums_to_206(char_data)`: Assert `sum(char_data["language_standards"]["distribution"].values()) == 206`
    - `test_language_standards_known_values(char_data)`: Assert distribution includes `"C++17": 128`, `"C++14": 51`, `"C11": 6`, `"C++11": 2`. (The 19 unspecified may be keyed as "unspecified" or "null".)
    - `test_language_standards_by_api_exists(char_data)`: Assert `"by_api"` key exists with api breakdown.

    **Section 8: Cross-metric consistency**
    - `test_total_kernels_consistent(char_data)`: Assert sloc kernel count == multi_file kernel count == 35.
    - `test_metadata_present(char_data)`: Assert `"metadata"` key with `"generated"`, `"script"`.
    - `test_md_output_exists()`: Assert `Path(PROJECT_ROOT / "results/analysis/benchmark_characterization.md").exists()`.

    **Total: ~25 tests covering all 6 CHAR requirements plus cross-metric validation.**
