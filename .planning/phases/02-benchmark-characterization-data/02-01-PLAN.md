    Create `scripts/analysis/benchmark_characterization.py` as a single monolithic script (per D-01).

    **Script structure:**

    1. **Imports and constants:**
       - Import `count_physical_sloc` from `sloc_analysis` (via sys.path.insert for scripts/analysis/)
       - Import `CORPUS_KERNELS` from `sloc_analysis`
       - Define `PROJECT_ROOT = Path(__file__).resolve().parent.parent.parent`
       - Accept `--project-root` (default: PROJECT_ROOT) and `--output-dir` (default: results/analysis/) via argparse

    2. **Function: `compute_sloc(project_root) -> dict`** (CHAR-01, per D-04, D-07)
       - Re-run SLoC analysis for all 35 CORPUS_KERNELS using `count_physical_sloc`
       - For each kernel, load CUDA spec from `specs/{suite}-{kernel}-cuda.json`
       - Count physical SLoC for each file in `prompt_payload`
       - Also compute OMP SLoC for translation size ratio (per D-04)
       - Compute aggregate stats: min, max, mean, median, std, distribution buckets (<100, 100-500, 500-1000, >1000)
       - Return dict with `per_kernel` (dict of kernel->data) and `summary` (aggregate stats)
       - Include `omp_cuda_ratio` per kernel where both exist: `omp_sloc / cuda_sloc`
       - Compute aggregate ratio stats: median ratio, min ratio, max ratio

    3. **Function: `compute_categories(project_root) -> dict`** (CHAR-02, per D-05)
       - Load manifest.jsonl, filter out phantom entries (check `Path(project_root / entry["spec_file"]).exists()`)
       - Build category -> {total_kernels: int, suites: {suite: count}} using DISTINCT kernel_name per category
       - Use `set()` to deduplicate kernel names within a category (a kernel appears in multiple API variants)
       - Return dict with 12 category keys, each having `kernel_count` and `suites` sub-dict
       - Example output: `{"graph": {"kernel_count": 5, "suites": {"rodinia": 3, "hecbench": 2}}}`

    4. **Function: `compute_api_coverage(project_root) -> dict`** (CHAR-03, per D-06)
       - Load manifest.jsonl, filter phantoms
       - Build suite x API matrix: for each (suite, api) pair, count DISTINCT kernel_names
       - Four API columns: cuda, omp, opencl, omp_target
       - Five suite rows: rodinia, hecbench, xsbench, rsbench, mixbench
       - Also compute column totals and row totals
       - Return dict: `{suites: {suite_name: {cuda: N, omp: N, opencl: N, omp_target: N, total: N}}, totals: {cuda: N, ...}}`

    5. **Function: `compute_multi_file(project_root) -> dict`** (CHAR-04, per D-02)
       - For each of the 35 CORPUS_KERNELS, load the CUDA spec
       - Record: `prompt_payload` count, `translation_targets` count
       - Classify: `translation_targets > 1` = multi-file, else single-file
       - Also load OMP, OpenCL specs if they exist for each kernel — record their target counts too
       - Build per_kernel dict and aggregate: `{single_file_count: N, multi_file_count: N, multi_file_pct: float}`
       - Also break down by suite and by API
       - The headline number for the paper is the multi-file percentage using `translation_targets > 1`

    6. **Function: `compute_language_features(project_root) -> dict`** (CHAR-05, per D-03)
       - For each CORPUS_KERNEL, find source files based on CUDA spec's `provenance.repo_root + provenance.source_path`
       - Read all source files in that directory (*.cu, *.c, *.cpp, *.h, *.hpp, *.cl)
       - Grep for feature indicators with these regex patterns:
         **OpenMP features** (applied to OMP spec source dirs):
         - Tier "omp_basic" (1.0): `#pragma\s+omp\s+parallel\s+for`
         - Tier "omp_4.5": `#pragma\s+omp\s+simd|#pragma\s+omp\s+taskloop`
         - Tier "omp_target" (4.5+): `#pragma\s+omp\s+target`
         **CUDA features** (applied to CUDA spec source dirs):
         - Tier "cuda_basic": `__global__|__shared__|__device__`
         - Tier "cuda_library": `thrust::|cub::`
         - Tier "cuda_9plus": `cooperative_groups`
         **OpenCL features** (applied to OpenCL spec source dirs):
         - Tier "opencl_1x": `clEnqueueNDRangeKernel|clCreateBuffer`
         - Tier "opencl_2x": `clSVMAlloc|pipe\s+`
       - For each kernel, record: `{features_found: [list], tier: "highest_tier_matched"}`
       - Tier assignment: highest tier found wins (e.g., if both `__global__` and `thrust::` found, tier = "cuda_library")
       - Handle missing source dirs gracefully (HeCBench is local/gitignored but IS available on this machine per CLAUDE.md: "HeCBench-master/ Gitignored but cloned locally (1874 benchmark dirs)")
       - Skip kernels whose source dir doesn't exist with a warning

    7. **Function: `compute_language_standards(project_root) -> dict`** (CHAR-06)
       - Iterate ALL 206 spec files in specs/
       - Extract `implementation.language_standard` (use `or "unspecified"` for None/missing)
       - Build distribution: `{standard: count}` — expected: C++11: 2, C++14: 51, C++17: 128, C11: 6, unspecified: 19
       - Also break down by API and by suite

    8. **Function: `main()`**
       - Parse args, resolve project_root
       - Call all 6 compute functions
       - Assemble combined JSON: `{metadata: {generated: ISO_timestamp, project_root: str, script: "benchmark_characterization.py"}, sloc: ..., categories: ..., api_coverage: ..., multi_file: ..., language_features: ..., language_standards: ..., summary: {total_kernels: 35, total_specs: 206, ...}}`
       - Write `benchmark_characterization.json` (JSON, indent=2)
       - Write `benchmark_characterization.md` with formatted tables for each section
       - Print summary to stdout
       - Return 0 on success

    9. **Markdown output** (benchmark_characterization.md):
       - Title: "# ParBench Benchmark Characterization"
       - Section per metric with markdown tables
       - SLoC table: kernel, suite, category, CUDA SLoC, OMP SLoC, ratio, src files, tgt files
       - Category table: category, kernel count, suites
       - API coverage cross-tab: suite x API matrix
       - Multi-file table: kernel, prompt_payload count, translation_targets count, classification
       - Language features: kernel, API, features found, tier
       - Language standards: standard, count, percentage
       - Summary statistics section at bottom

    **Critical implementation details:**
    - Use `sys.path.insert(0, str(Path(__file__).resolve().parent))` to import from sloc_analysis
    - Filter manifest.jsonl phantoms: `Path(project_root / entry["spec_file"]).exists()`
    - Use `(spec.get("implementation") or {}).get("language_standard")` for null-safe access (per project convention)
    - Handle missing source dirs: print warning to stderr, skip that kernel's language features, but still include it in SLoC/multi-file from spec data
    - OMP/CUDA ratio: `round(omp_sloc / cuda_sloc, 2)` where both exist; `null` otherwise
    - All counts must be integers, all percentages `round(x, 1)` floats
