    **Script structure:**

    ```python
    #!/usr/bin/env python3
    """
    scripts/analysis/validate_characterization.py

    Independent cross-validation of benchmark_characterization.json against raw data sources.
    Uses a completely different code path from benchmark_characterization.py to catch bugs.

    Checks:
      1. SLoC: all 35 kernels present, SLoC values > 0
      2. Categories: 12 categories, kernel counts match manifest.jsonl
      3. API coverage: 5x4 matrix matches spec file enumeration
      4. Multi-file: counts match spec prompt_payload/translation_targets
      5. Language features: valid tiers, coverage of available source dirs
      6. Language standards: distribution sums to 206, matches spec field scan
      7. Cross-metric: kernel counts consistent across sections

    Usage:
        python3 scripts/analysis/validate_characterization.py \\
            --project-root /home/samyak/Desktop/parbench_sam
    """
    ```

    **Imports:**
    - `argparse`, `json`, `sys`, `collections.defaultdict`, `pathlib.Path`
    - `sys.path.insert(0, str(Path(__file__).resolve().parent))` for importing `CORPUS_KERNELS` from `sloc_analysis`

    **Architecture: Check functions return `list[str]` (empty = pass, non-empty = failure messages)**

    **Function: `load_characterization(project_root) -> dict`**
    - Load `results/analysis/benchmark_characterization.json`
    - If file doesn't exist, print error and sys.exit(1) with message: "Run benchmark_characterization.py first"

    **Function: `load_valid_manifest(project_root) -> list[dict]`**
    - Parse manifest.jsonl
    - Filter: only entries where `Path(project_root / entry["spec_file"]).exists()`
    - Return list of valid entries (expected: 206)

    **Function: `check_schema(data) -> list[str]`**
    - Verify top-level keys: `metadata`, `sloc`, `categories`, `api_coverage`, `multi_file`, `language_features`, `language_standards`, `summary`
    - Verify `metadata` has `generated` and `script` keys
    - Return list of missing/unexpected key errors

    **Function: `check_sloc(data, project_root) -> list[str]`** (validates CHAR-01)
    - Import CORPUS_KERNELS from sloc_analysis
    - Verify `len(data["sloc"]["per_kernel"]) == 35`
    - Verify set of kernel names matches `{k for _, k in CORPUS_KERNELS}`
    - Verify each kernel has `physical_sloc > 0`
    - Verify summary has `min_sloc`, `max_sloc`, `mean_sloc`, `median_sloc`
    - Verify `min_sloc <= median_sloc <= max_sloc`
    - Verify OMP/CUDA ratio present for at least some kernels (D-04)
    - Return list of errors

    **Function: `check_categories(data, manifest_entries) -> list[str]`** (validates CHAR-02)
    - Independently compute: for each valid manifest entry, collect `(kernel_name, category, source_suite)` tuples
    - Deduplicate by kernel_name within each category (kernel appears in multiple APIs)
    - Build expected: `{category: {kernel_count: N, suites: {suite: count}}}`
    - Compare expected vs `data["categories"]` — exact match on kernel_count per category
    - Verify exactly 12 categories present
    - Verify each category has `suites` sub-dict (D-05)
    - Return list of mismatches

    **Function: `check_api_coverage(data, manifest_entries) -> list[str]`** (validates CHAR-03)
    - Independently compute: for each valid manifest entry, count distinct kernel_names per (source_suite, parallel_api)
    - Build expected 5x4 matrix
    - Compare cell by cell against `data["api_coverage"]["suites"]`
    - Verify totals are correct column sums
    - Verify 4 API columns per D-06: cuda, omp, opencl, omp_target
    - Return list of mismatches with expected vs actual

    **Function: `check_multi_file(data, project_root) -> list[str]`** (validates CHAR-04)
    - For each of the 35 corpus kernels, load CUDA spec
    - Independently count `prompt_payload` and `translation_targets`
    - Compare against `data["multi_file"]["by_kernel"]`
    - Verify classification: `is_multi_file == (translation_targets_count > 1)` per D-02
    - Verify aggregate: `single_file_count + multi_file_count == 35`
    - Return list of mismatches

    **Function: `check_language_features(data) -> list[str]`** (validates CHAR-05)
    - Verify `per_kernel` key exists
    - Verify all tier values are from valid set: `{"cuda_basic", "cuda_library", "cuda_9plus", "omp_basic", "omp_4.5", "omp_target", "opencl_1x", "opencl_2x", "undetected"}`
    - Verify at least 25 kernels have entries (some may have missing source dirs)
    - Return list of invalid tier values

    **Function: `check_language_standards(data, project_root) -> list[str]`** (validates CHAR-06)
    - Independently scan ALL spec files in `specs/`
    - Extract `implementation.language_standard` (None -> "unspecified")
    - Build expected distribution
    - Compare against `data["language_standards"]["distribution"]`
    - Verify total sums to 206
    - Return list of mismatches

    **Function: `check_cross_metric(data) -> list[str]`**
    - Verify sloc kernel count == multi_file kernel count
    - Verify category kernel count total >= 35 (categories cover corpus kernels)
    - Verify api_coverage total specs ~= 206
    - Return list of inconsistencies

    **Function: `main()`**
    - Parse --project-root arg
    - Load characterization data
    - Load manifest entries
    - Run all 8 check functions
    - Print results in clear format:
      ```
      === Benchmark Characterization Validation ===

      [PASS] Schema check (8/8 required keys present)
      [PASS] SLoC (CHAR-01): 35/35 kernels, all SLoC > 0
      [FAIL] Categories (CHAR-02): expected 5 kernels in 'graph', got 4
        - graph: expected kernel_count=5, got 4
      [PASS] API coverage (CHAR-03): 5x4 matrix matches manifest
      ...

      RESULT: 7/8 checks passed, 1 FAILED
      ```
    - Exit 0 if all pass, exit 1 if any fail
    - Write validation report to `results/analysis/characterization_validation.txt`

    **Critical implementation details:**
    - COMPLETELY independent code path: do NOT import anything from benchmark_characterization.py
    - Import only CORPUS_KERNELS from sloc_analysis.py (for the kernel list, which is ground truth)
    - Use `(spec.get("implementation") or {}).get("language_standard")` for null-safe access
    - Filter manifest phantoms by checking `Path(project_root / entry["spec_file"]).exists()`
    - Print each check's status immediately (not buffered) so the user sees progress
    - Use structured error messages: `f"{section}: expected {expected}, got {actual}"`
