    **Step 2: Create augmentation_analysis.py** (GREEN phase)

    Create `scripts/analysis/augmentation_analysis.py` following the Phase 2 `benchmark_characterization.py` monolithic pattern (per D-13). The script must:

    1. **Parse args**: `--project-root` (required), `--output-dir` (default: `results/analysis/`), `-v` verbose flag. Use `argparse`.

    2. **Define constants**:
       - `KNOWN_FAIL_SOURCES = frozenset({"rodinia-kmeans-cuda", "rodinia-mummergpu-cuda"})` — source specs whose CUDA source is KNOWN_FAIL
       - `EVAL_DIR_NAME = "together-qwen-3.5-397b-a17b"`
       - Direction detection from filename: parse `{suite}-{kernel}-{src_api}-to-{suite}-{kernel}-{tgt_api}` pattern

    3. **`_augment_level_from_filename(stem: str) -> int`**: Extract level from filename. Convention: `-L{N}` suffix -> N, no suffix -> 0. Filter out stochastic samples (`-s\d+`).

    4. **`_parse_result_filename(stem: str) -> dict | None`**: Parse a result filename into `{"source_spec", "target_spec", "kernel", "source_api", "target_api", "direction", "level"}`. Handle the compound format: `{suite}-{kernel}-{src_api}-to-{suite}-{kernel}-{tgt_api}[-L{N}][-s{M}]`. Return None for unparseable filenames or stochastic samples.

       CRITICAL: When checking for `omp` vs `omp_target`, check for `omp_target` FIRST (since `omp` is a substring). The target API is the last API segment before `-L` or end-of-stem. Use the target spec's JSON fields (`target_spec`) when available from the result JSON itself, falling back to filename parsing only when needed.

    5. **`build_primary_matrix(results_dir: Path) -> dict`**: Build cuda-to-omp matrix.
       - Glob all `*.json` files in results_dir
       - For each file: parse filename, skip stochastic samples, skip non-cuda-to-omp directions (and skip cuda-to-omp_target)
       - Load JSON, extract `overall_status` (CRITICAL: use `overall_status`, never `run_status`)
       - Group by kernel, store L0-L4 statuses
       - Mark known_fail based on source_spec
       - Return dict with per_kernel entries

    6. **`classify_patterns(per_kernel: dict) -> dict`**: For each kernel:
       - If all L0-L4 are PASS -> `stable_pass`
       - If all L0-L4 are non-PASS -> `stable_fail`
       - If L0 is PASS and any L1-L4 is non-PASS -> `degradation`
       - If L0 is non-PASS and any L1-L4 is PASS -> `improvement`
       - Otherwise -> `other`
       - Return `{"stable_pass": [...], "stable_fail": [...], "degradation": [...], "improvement": [...], "other": [...]}`

    7. **`compute_aggregates(per_kernel: dict) -> dict`**: For each level L0-L4:
       - Count total kernels and PASS kernels
       - Compute Wilson 95% CI using the same formula from `statistical_analysis.py` (re-implement inline to avoid import complexity, or import if clean)
       - Also compute aggregate excluding KNOWN_FAIL

    8. **`identify_exceptions(per_kernel: dict) -> list`**: For each degradation and improvement kernel, build an exception entry with: kernel name, pattern type, L0 status, affected levels with their statuses, root cause placeholder (filled for backprop: "Linker error: multiple definition of 'gettime' and 'main' — LLM duplicated functions across files when given augmented source. Augmentation engine validated separately (backprop-cuda PASSES at all harness levels). This is genuine model brittleness, not a transform artifact.").

    9. **`build_secondary_matrix(results_dir: Path) -> dict`**: Build aggregate pass rates for ALL directions (not just cuda-to-omp). For each direction found in the data, compute per-level aggregates with Wilson CIs. Group by direction string. Include at least these 7 directions: cuda-to-omp, omp-to-cuda, cuda-to-opencl, opencl-to-cuda, omp-to-opencl, opencl-to-omp, omp_target-to-cuda.

    10. **`generate_markdown(data: dict) -> str`**: Produce a markdown summary containing:
        - "# Per-Kernel Augmentation Matrix" header
        - Per-kernel table (kernel | suite | L0 | L1 | L2 | L3 | L4 | pattern)
        - "## Pattern Summary" section with counts for each pattern type
        - "## Aggregate Pass Rates" section with level | pass | total | rate | CI
        - "## Exceptions" section listing each exception with root cause
        - "## Secondary Directions" section with per-direction aggregate table

    11. **`main() -> int`**: Orchestrate all computation:
        - Parse args
        - Build primary matrix
        - Classify patterns
        - Compute aggregates (full + excluding KNOWN_FAIL)
        - Identify exceptions
        - Build secondary matrix
        - Assemble combined JSON (schema from 03-RESEARCH.md JSON Schema Recommendation)
        - Write JSON to `results/analysis/augmentation_per_kernel_matrix.json`
        - Generate and write markdown to `results/analysis/augmentation_per_kernel_matrix.md`
        - Return 0
