## Twitter/X

- **Max length:** 280 chars (threads: 10 tweets max)
- **Tone:** Punchy, opinionated, technical. No hashtag stuffing.
- **Format:** One idea per tweet. If it needs explanation, make it a thread.
- **Media:** Code screenshots > text-only. Short GIFs for demos.

## LinkedIn

- **Max length:** 3,000 chars (but front-load value — most won't expand)
- **Tone:** Professional but not corporate. Still our voice, just slightly more formal.
- **Format:** Hook in first line. Whitespace between paragraphs. Use line breaks aggressively.
- **Media:** Carousels perform well. PDF slide decks as carousels.

## Newsletter (Email)

- **Length:** 500-800 words. Scannable. 3-5 sections max.
- **Tone:** Like a smart colleague's weekly email. Personal but valuable.
- **Format:** TL;DR at top, then sections with headers.
- **CTA:** One primary CTA. Don't compete for attention.

## Dev.to / Hashnode / Medium

- **Length:** Full blog post (repost from writing-room with minor adaptation)
- **Tone:** Same as blog
- **Format:** Add platform-appropriate tags. Adjust code examples if needed.
- **Note:** Always canonical URL back to original blog post for SEO
