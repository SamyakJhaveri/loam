\begin{abstract}
We present \parbench{}, a kernel-centric benchmark framework for evaluating LLM-based parallel code translation across CUDA, OpenMP, and OpenCL. \parbench{} combines declarative benchmark specifications, an end-to-end build-run-verify harness, survey-grounded corpus construction from 35~open-source HPC repositories, and AST-driven semantics-preserving source augmentation. The corpus contains 96~specifications over 35~kernels from five benchmark suites, yielding six standard translation directions plus two OpenMP-target case-study directions. Evaluated under stochastic sampling (temperature~0.7, three independent attempts per task), \qwen{} achieves a 36.7\%~[33.1\%, 40.6\%] overall pass rate across 626~valid translation records, with pass@1\,=\,23.9\% and pass@3\,=\,35.2\% over 142 unique source-target pairs; \gptnew{} evaluation is in progress (\tbd{}). Build-stage adaptation failures account for 39.1\% of all outcomes, making incomplete API surface mapping the dominant bottleneck, while verified semantic failures constitute 4.6\%. On a balanced 12-kernel CUDA-to-OpenMP subset present at all five augmentation levels, pass rates remain stable (75--83\%), providing no evidence that baseline success depends on surface-form memorization. \parbench{} exposes both non-trivial current capability and the specific structural barriers---direction asymmetry, multi-file coordination, and incomplete API adaptation---that remain for reliable parallel code translation.
\end{abstract}
