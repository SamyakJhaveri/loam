\begin{abstract}
We present \parbench{}, a kernel-centric benchmark framework for evaluating LLM-based parallel code translation across multiple APIs.\parbench{} combines declarative benchmark specifications, an end-to-end build-run-verify harness, survey-grounded corpus construction from 35~open-source HPC repositories, and AST-driven semantics-preserving source augmentation. The corpus contains 96~specifications over 35~kernels from five benchmark suites, yielding six standard translation directions plus two OpenMP-target case-study directions. \gptnew{} achieves a 62.7\%~[58.0\%, 67.1\%] L0 pass rate with pass@1\,=\,62.7\% and pass@3\,=\,69.7\% over the same 142~tasks. For \qwenshort{}, build-stage adaptation failures account for 39.1\% of outcomes, making incomplete API surface mapping the dominant bottleneck, while verified semantic failures constitute 4.6\%. On a balanced 12-kernel CUDA-to-OpenMP subset present at all five augmentation levels, pass rates remain stable (75--83\%), providing no evidence that baseline success depends on surface-form memorization. \parbench{} exposes both non-trivial current capability and the specific structural barriers---direction asymmetry, multi-file coordination, and incomplete API adaptation that remain for reliable parallel code translation.
\end{abstract}
