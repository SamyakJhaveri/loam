## 6. Recovery and rollback

**If a stage's verification fails 3 times in a row** — per `.claude/rules/stage-contract.md`, the issue is the contract, not the implementation. Stop. Write `docs/multi_session_prompts/2026-05-17-v2.0-stage-N-blocker.md` describing what's blocking, what was tried, and what design call is needed. Do not keep iterating.

**If a later stage exposes a defect in an earlier stage** — prefer fix-forward (a new commit) over `git revert`. Never use `git reset --hard` past a committed stage; the per-stage commits exist to make incremental review possible.

**If a `git mv` batch partial-fails mid-stage (Stage 5 specifically)** — `git reset --hard HEAD` to restore the pre-stage state, identify the cause (commonly an untracked file in a moved dir), resolve, and retry. Do not commit a half-moved tree.

**If `/plugin install` or `/plugin marketplace add` fails in Stage 5 or 7** — the most common cause is a missing required field in `.claude-plugin/plugin.json` or `.claude-plugin/marketplace.json`. Re-read the Anthropic plugin docs URL and compare against `anthropics/skills` or `obra/superpowers` for canonical field shape.
