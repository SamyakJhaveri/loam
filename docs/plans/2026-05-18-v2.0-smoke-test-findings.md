## Finding: `--vcs-ref HEAD` is required for local Copier copies

Copier 9.15.1 in "dirty mode" (no `--vcs-ref`) skips some files when copying from a local git repo:
- `.claude/hooks/session-start.sh`
- `.claude/hooks/codex-review-reminder.sh`
- `.claude/hooks/test-bash-audit-log.sh`

These files are tracked in git and contain no Jinja syntax. The cause is unclear — likely a Copier heuristic for non-VCS-ref local copies. Adding `--vcs-ref HEAD` resolves the issue.

`bin/verify-template.sh` already uses `--vcs-ref HEAD` via `COPIER_FLAGS`, so existing CI/verification is correct. End-user `copier copy` from a GitHub URL is unaffected (remote copies always use a ref).

**Action**: No code change needed. `--vcs-ref HEAD` is already the project default.
